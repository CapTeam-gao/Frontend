# CapTeam

AI 기반 캡스톤 팀 자동생성 및 관리 올인원 서비스 **CapTeam**입니다.

CapTeam은 캡스톤 프로젝트에서 발생하는 팀 구성과 운영의 불편함을 줄이기 위해 기획된 서비스입니다.

학생 설문 데이터를 기반으로 AI가 팀 추천안을 생성하고, 관리자가 추천안을 검토/수정/승인한 뒤 공지, 채팅, 프로젝트 기획서, 캡스톤 일지까지 하나의 서비스에서 관리할 수 있도록 합니다.

## 프로젝트 소개

캡스톤 팀 구성은 단순히 인원만 나누는 작업이 아닙니다.

학생의 희망 직군, 기술 스택, 구현 경험, 협업 성향, 선호 팀원, 팀장 희망 여부 등을 함께 고려해야 합니다.

기존에는 이 과정을 수동으로 처리해야 했고, 공지 작성, 팀 관리, 일지 수집, 프로젝트 정보 관리도 여러 플랫폼에 흩어져 있어 운영이 불편했습니다.

CapTeam은 이런 과정을 자동화하고 통합하는 것을 목표로 합니다.

## 핵심 목표

- 학생 설문 기반 팀 매칭 정보 수집
- AI 기반 캡스톤 팀 추천
- 관리자 검토/수정/승인 가능
- 팀 생성 이후 프로젝트 운영 기능 제공
- 공지, 채팅, 프로젝트 기획서, 캡스톤 일지 통합 관리

## 사용자 흐름

### 학생

```txt
로그인
 → 설문 미완료 시 설문 작성
 → 공지 확인
 → 팀 생성 완료 후 프로젝트 기능 사용
 → 팀 채팅
 → 프로젝트 기획서 작성
 → 캡스톤 일지 작성
```

### 관리자

```txt
로그인
 → 학생 설문 완료 여부 확인
 → 학년별 팀 생성 요청
 → AI 추천 팀 검토
 → 팀원 수동 교체
 → 최종 승인
 → 팀 관리
 → 공지 관리
 → 캡스톤 일지 관리
 → 채팅 관리
```

## 주요 기능

### AI 팀 생성

- 학생 설문 데이터 분석
- 기술 스택, 구현 경험, 희망 직군 기반 역량 분석
- 성격 성향 / 개발 성향 점수 반영
- 역할군 분산
- 팀 간 실력 균형 고려
- 선호 팀원 및 팀장 희망 여부 반영
- LLM 보정
- 중복/누락/인원 차이 검증
- 팀별 배정 이유 제공

### 관리자 기능

- 관리자 대시보드
- 학생 관리
- 학생 설문 상세 조회
- 팀 생성 요청
- 팀 추천안 검토
- 팀원 수동 교체
- 팀 추천안 승인
- 확정 팀 관리
- 공지 CRUD
- 캡스톤 일지 제출 현황 관리
- 팀별 채팅 관리

### 학생 기능

- 팀 매칭 설문 작성
- 공지 조회
- 팀 채팅
- 프로젝트 기획서 작성/수정
- 캡스톤 일지 작성/조회
- 내 정보 관리

### 채팅 기능

- STOMP 기반 실시간 채팅
- 팀별 채팅방
- 채널 생성/수정/삭제
- 팀장 권한 기반 채널 관리
- 메시지 전송/수정/삭제
- 파일 업로드/다운로드
- 이미지 미리보기
- 링크 자동 감지
- 온라인/오프라인 표시
- 읽지 않은 채팅 표시

## 시스템 구조

```txt
Frontend
  ├─ 학생 화면
  ├─ 관리자 화면
  ├─ REST API 연동
  └─ WebSocket 채팅 연동

Backend
  ├─ 인증/인가
  ├─ 학생/팀/공지/일지 API
  ├─ 채팅 API
  ├─ WebSocket 이벤트 처리
  └─ AI 서버 연동

AI Server
  ├─ 학생 분석
  ├─ 1차 팀 생성
  ├─ LLM 보정
  ├─ 검증
  └─ 최종 추천 결과 반환
```

## AI 팀 배정 흐름

```txt
학생 설문 데이터 수집
 → 학생별 역량 분석
 → 기술 점수 / 성향 점수 계산
 → 규칙 기반 1차 팀 생성
 → LLM 보정
 → 중복/누락/인원 차이/역할 쏠림 검증
 → 최종 추천 팀 반환
 → 백엔드 추천안 저장
 → 관리자 검토/수정/승인
 → 최종 팀 확정
```

## 프로젝트 특징

- AI가 생성한 팀을 바로 확정하지 않고 관리자가 검토할 수 있습니다.
- 팀별 배정 이유를 함께 제공하여 팀 구성의 설명 가능성을 높였습니다.
- 기술 스택뿐 아니라 성향 점수를 함께 반영합니다.
- 학생 설문, 팀 생성, 공지, 일지, 채팅을 하나의 흐름으로 연결했습니다.
- 프론트엔드, 백엔드, AI 서버가 분리된 구조로 개발되었습니다.

## 향후 개선 사항

- 설문 문항 신뢰도 개선
- 역문항과 응답 일관성 기반 신뢰도 계산 고도화
- AI 팀 재생성 프롬프트 기능 개선
- 팀 배정 이유 품질 개선
- 배포 환경 안정화
- 실제 학생 데이터 기반 QA 진행
