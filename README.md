# CapTeam Frontend

CapTeam은 캡스톤 프로젝트의 팀 구성부터 운영까지 한곳에서 관리하기 위한 AI 기반 캡스톤 팀 자동생성 및 관리 서비스입니다.

프론트엔드는 학생과 관리자의 사용 흐름을 분리하고, 로그인 이후 권한과 설문 완료 여부, 팀 생성 상태에 따라 필요한 화면으로 이동할 수 있도록 구성했습니다.

## 프로젝트 목적

캡스톤 팀 구성은 학생의 희망 직군, 기술 스택, 구현 경험, 협업 성향, 선호 팀원 등을 함께 고려해야 하므로 수작업으로 처리하기 어렵습니다.

CapTeam은 학생 설문 데이터를 기반으로 팀 매칭에 필요한 정보를 수집하고, 관리자가 학년별 팀 생성을 요청한 뒤 생성된 추천안을 검토, 수정, 승인할 수 있는 흐름을 제공합니다.

팀 확정 이후에는 공지, 팀 관리, 채팅, 프로젝트 기획서, 캡스톤 일지 기능으로 프로젝트 운영까지 이어지는 구조를 목표로 합니다.

## 핵심 사용자

| 사용자 | 주요 역할 |
| --- | --- |
| 학생 | 설문 작성, 공지 확인, 팀 채팅, 프로젝트 기획서 작성, 캡스톤 일지 작성 |
| 관리자 | 학생 설문 확인, 팀 생성 요청, 팀 추천안 검토/수정/승인, 학생 관리, 공지 관리, 일지 관리 |

## 서비스 흐름

```txt
로그인
 ├─ 학생 계정
 │   ├─ 설문 안내
 │   ├─ 팀 매칭 설문 작성
 │   ├─ 학생 대시보드
 │   ├─ 공지 확인
 │   ├─ 팀 생성 이후 프로젝트 기능 접근
 │   ├─ 프로젝트 기획서 작성
 │   ├─ 캡스톤 일지 작성
 │   └─ 팀 채팅
 │
 └─ 관리자 계정
     ├─ 관리자 대시보드
     ├─ 학생 설문 완료 여부 확인
     ├─ 학년별 팀 생성 요청
     ├─ 팀 생성 로딩
     ├─ 팀 추천안 검토
     ├─ 학생 선택 기반 팀원 교체
     ├─ 팀 추천안 승인
     ├─ 팀 관리
     ├─ 공지 작성/수정/삭제
     └─ 캡스톤 일지 관리
```

## 주요 기능

### 인증 및 전역 상태

- JWT 기반 로그인 흐름을 기준으로 `accessToken`과 사용자 정보를 관리합니다.
- Zustand 기반 `authStore`로 로그인 상태와 사용자 정보를 전역에서 사용합니다.
- Axios 인스턴스에서 요청 인터셉터를 통해 `Authorization: Bearer {token}` 헤더를 자동으로 붙입니다.
- 401 응답 발생 시 기본적으로 로그인 페이지로 이동하며, 일부 상세 조회 요청은 `skipAuthRedirect` 옵션으로 화면 내 에러 처리를 우선합니다.

### 학생 설문

- 학생 로그인 후 팀 매칭에 필요한 설문을 작성하는 흐름을 제공합니다.
- 희망 직군, 기술 스택, 구현 경험, 선호 팀원, 팀장 선호 여부를 입력받습니다.
- 성격 성향과 개발 성향 문항을 각각 5개 기준으로 나누고, 기준별 점수를 계산해 백엔드로 전달합니다.
- 설문 완료 전에는 학생 대시보드의 프로젝트 관련 기능 접근을 제한합니다.

### 관리자 학생 관리

- 전체 학생 수, 학년별 학생 수, 설문 미제출 학생 수를 확인할 수 있습니다.
- 이름, 학번, 희망 직군 기준으로 학생을 검색할 수 있습니다.
- 학생 카드를 클릭하면 모달에서 설문 입력 정보, 기술 스택, 구현 경험, 선호 팀원, 분석 결과를 확인할 수 있습니다.
- 학생 카드와 학생 상세 모달은 별도 컴포넌트로 분리했습니다.

### 팀 생성 및 팀 관리

- 관리자는 학년을 선택해 팀 추천안 생성을 요청할 수 있습니다.
- 팀 생성 요청 이후 로딩 페이지를 통해 진행 상태를 표시합니다.
- 추천안 목록에서 팀별 역할 분포와 배정 이유를 확인할 수 있습니다.
- 학생 2명을 선택해 팀원을 교체하는 수동 수정 흐름을 제공합니다.
- 추천안을 승인하면 실제 팀/팀원 데이터가 생성되고, 팀 관리 페이지에서 확정된 팀을 조회합니다.

### 공지 관리

- 관리자 공지 작성, 목록 조회, 상세 조회, 수정, 삭제 기능을 백엔드 API와 연동했습니다.
- 공지 작성/수정 화면에는 마크다운 에디터를 적용했습니다.
- 학생은 공지 목록과 상세 내용을 확인할 수 있습니다.
- 새 공지가 등록되면 학생 대시보드에서 읽지 않은 공지 표시를 갱신할 수 있도록 이벤트 기반 갱신 흐름을 구성했습니다.

### 대시보드

- 관리자 대시보드는 팀 생성 전/후 상태에 따라 다른 정보를 보여줍니다.
- 팀 생성 전에는 팀 생성 기능을 강조하고, 팀 생성 이후에는 전체 팀 수와 학년별 팀 수를 표시합니다.
- 학생 대시보드는 팀 채팅, 프로젝트, 캡스톤 일지, 공지 진입점을 제공합니다.
- 팀 생성 전에는 학생 프로젝트 기능을 막고, 팀 생성 이후에 이동할 수 있도록 처리했습니다.

### 프로젝트 기획서 및 캡스톤 일지

- 프로젝트 기획서 입력 화면은 퍼블리싱과 기본 입력 흐름을 구현했습니다.
- 캡스톤 일지 작성/결과 화면과 관리자 일지 목록/상세 화면을 구성했습니다.
- 프로젝트 기획서와 캡스톤 일지의 일부 API 연동은 최종 개발 단계에서 확장 예정입니다.

## 최종 서비스 방향

CapTeam은 팀을 자동으로 나누는 기능에서 끝나는 서비스가 아니라, 캡스톤 프로젝트 운영 흐름 전체를 관리하는 서비스를 목표로 합니다.

학생은 설문을 통해 자신의 기술과 협업 성향을 입력하고, 관리자는 추천된 팀을 검토하고 수정한 뒤 승인할 수 있습니다.

이후 공지, 일지, 채팅, 프로젝트 기획서 기능을 통해 팀 생성 이후의 프로젝트 운영까지 이어지는 구조를 제공합니다.
