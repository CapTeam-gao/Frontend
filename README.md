# CapTeam Frontend

AI 기반 캡스톤 팀 자동생성 및 관리 올인원 서비스 **CapTeam**의 프론트엔드 레포지토리입니다.

CapTeam은 학생 설문 데이터를 기반으로 캡스톤 팀을 추천하고, 관리자가 추천 팀을 검토/수정/승인한 뒤 프로젝트 운영까지 이어갈 수 있도록 돕는 서비스입니다.

프론트엔드는 학생과 관리자 화면을 분리하여 구현했으며, 로그인 권한, 설문 완료 여부, 팀 생성 상태에 따라 다른 화면 흐름을 제공합니다.

## 프로젝트 목적

캡스톤 팀 구성 과정에서는 학생의 희망 직군, 기술 스택, 구현 경험, 협업 성향, 선호 팀원, 팀장 희망 여부 등 여러 요소를 함께 고려해야 합니다.

기존에는 이 과정을 수동으로 처리해야 해서 시간이 오래 걸리고, 공지 작성, 일지 수집, 팀 관리 등도 여러 플랫폼에 나뉘어 있어 불편함이 있었습니다.

CapTeam은 이 과정을 하나의 서비스 안에서 처리할 수 있도록 기획했습니다.

## 주요 기능

### 학생 기능

-   로그인
-   팀 매칭 설문 작성
-   희망 직군, 기술 스택, 구현 경험 입력
-   성격 성향 / 개발 성향 설문 응답
-   선호 팀원 및 팀장 희망 여부 입력
-   학생 대시보드
-   공지 목록 및 상세 조회
-   프로젝트 기획서 작성 / 조회 / 수정
-   캡스톤 일지 작성 / 조회
-   팀 채팅
-   파일 업로드 / 이미지 미리보기 / 링크 이동
-   내 정보 및 비밀번호 변경

### 관리자 기능

-   로그인
-   관리자 대시보드
-   학년별 팀 생성 요청
-   AI 팀 추천안 검토
-   팀원 수동 교체
-   팀 추천안 승인
-   확정 팀 관리
-   팀 상세 모달 조회
-   학생 관리
-   학생 설문 상세 조회
-   개발 성향 / 성격 성향 차트 조회
-   공지 작성 / 조회 / 수정 / 삭제
-   캡스톤 일지 제출 현황 관리
-   캡스톤 일지 상세 조회
-   관리자 채팅 관리
-   내 정보 및 비밀번호 변경

## 서비스 흐름

```txt
학생
로그인
 → 설문 미완료 시 설문 페이지 이동
 → 설문 작성
 → 대시보드
 → 팀 생성 전에는 프로젝트 기능 제한
 → 팀 생성 후 팀 채팅 / 프로젝트 기획서 / 캡스톤 일지 사용

관리자
로그인
 → 학생 설문 완료 여부 확인
 → 학년별 팀 생성
 → AI 추천 팀 검토
 → 필요 시 팀원 수동 교체
 → 팀 승인
 → 팀 관리 / 공지 / 일지 / 채팅 관리
```

## 기술 스택

| 구분      | 기술                 |
| --------- | -------------------- |
| Framework | React 19, Vite       |
| Routing   | React Router         |
| State     | Zustand              |
| API       | Axios                |
| Realtime  | STOMP, SockJS        |
| Chart     | Recharts             |
| Editor    | @uiw/react-md-editor |
| Styling   | CSS Modules          |
| Deploy    | Vercel               |

## 폴더 구조

```txt
src
 ├─ api             # 기능별 API 요청 함수
 ├─ assets          # 이미지, 아이콘 파일
 ├─ components      # 공통/관리자/학생 컴포넌트
 │   ├─ admin
 │   ├─ common
 │   └─ user
 ├─ constants       # 고정 데이터, 설문 문항, 역할 기준
 ├─ hooks           # 커스텀 훅
 ├─ pages           # 라우터에 연결되는 페이지
 │   ├─ admin
 │   ├─ auth
 │   └─ user
 ├─ router          # 라우터 설정
 ├─ store           # Zustand 전역 상태
 ├─ styles          # 전역 CSS
 └─ utils           # 포맷팅, 검증, 데이터 가공 함수
```

## 주요 구현 내용

### 인증

-   JWT 기반 로그인 처리
-   `accessToken`을 Zustand와 localStorage에 저장
-   Axios interceptor를 통해 API 요청 시 자동으로 Authorization 헤더 추가
-   accessToken 만료 시 refreshToken 기반 재발급 처리
-   권한에 따라 관리자/학생 페이지 분리

### 설문

-   성격 성향 5개 기준, 개발 성향 5개 기준으로 문항 구성
-   기준별 평균 점수 계산
-   역문항과 응답 신뢰도 계산 구조 추가
-   설문 완료 후 학생 분석 및 팀 생성에 활용할 데이터 전송

### 팀 생성

-   학년별 팀 생성 요청
-   AI 추천 결과 조회
-   팀별 역할 분포 및 배정 이유 확인
-   학생 선택 기반 팀원 교체
-   추천안 최종 승인
-   팀 생성 완료 후 대시보드와 헤더 상태 변경

### 학생 관리

-   학생 수, 학년별 학생 수, 설문 미제출 학생 확인
-   이름, 학번, 희망 직군 검색
-   학생 상세 모달
-   기술 스택, 구현 경험, 선호 팀원, 분석 결과 조회
-   개발 성향 / 성격 성향 방사형 차트 표시
-   응답 신뢰도 UI 표시

### 프로젝트 기획서

-   팀명, 서비스명, 서비스 소개, 주요 기능 입력
-   주요 기능 동적 추가/삭제
-   저장된 내용 조회 및 수정
-   팀 관리 페이지와 연동

### 캡스톤 일지

-   학생 일지 작성
-   팀장/팀원 역할에 따른 작성 폼 분리
-   작성 가능 시간 카운트다운
-   기존 작성 내용 수정
-   완료된 일지 수정 제한
-   관리자 일지 목록/상세 조회

### 채팅

-   STOMP + SockJS 기반 실시간 채팅
-   학생 팀 채팅
-   관리자 팀별 채팅 관리
-   채널 생성/수정/삭제
-   팀장만 채널 관리 가능
-   메시지 전송/수정/삭제
-   파일 업로드 및 다운로드
-   이미지 미리보기
-   링크 자동 감지
-   읽지 않은 채팅 표시
-   온라인/오프라인 상태 표시
